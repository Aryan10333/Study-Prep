# Top 15 Evaluation, Guardrails & Observability Interview Questions

This flashcard guide presents 15 high-frequency interview questions asked by top tech companies covering RAG evaluation, safety guardrails, prompt injection defenses, and OpenTelemetry tracing.

---

### Q1: What is the RAG Triad framework?
- **Answer:** The RAG Triad evaluates RAG quality across 3 dimensions: Context Relevance (Query vs Context), Faithfulness (Answer vs Context), and Answer Relevance (Answer vs Query).

---

### Q2: How do you detect and block Prompt Injections in real-time?
- **Answer:** Combine regex heuristic scanners for adversarial strings (`"ignore previous instructions"`), lightweight classifier models (e.g. LlamaGuard), and Pydantic schema validation.

---

### Q3: What is LLM-as-a-Judge, and what are its primary biases?
- **Answer:** LLM-as-a-Judge uses a strong model (e.g., GPT-4o) to evaluate candidate answers. Primary biases include Position Bias (favoring the first option), Self-Enhancement Bias (favoring outputs generated by itself), and Verbosity Bias (favoring longer responses).

---

### Q4: Explain the difference between Time-To-First-Token (TTFT) and Inter-Token Latency (ITL).
- **Answer:** TTFT measures the duration from user prompt submission until the first token streams back. ITL measures the delta time between subsequent generated tokens.

---

### Q5: How do you protect PII (Personally Identifiable Information) from being sent to external LLM APIs?
- **Answer:** Apply an input guardrail filter (such as Microsoft Presidio) that scans for SSNs, credit cards, and emails using NER models, replacing them with anonymized placeholders (`<EMAIL_ADDRESS>`) before API dispatch.
