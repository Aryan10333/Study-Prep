# Workspace Customization Rules: DS/AI Interview Prep for GenAI/Agentic AI Engineers

This document outlines the background and constraints for the AI agent assisting with study prep.

---

## 1. User Profile and Context

- **Experience Level:** AI Engineer with ~3 years of experience.
- **Primary Domain:** GenAI architectures (LLMs, RAG, prompt engineering, vector databases) and Agentic AI solutions (multi-agent orchestration, tool usage, planning, reflection).
- **Goal:** Prep for Data Science (DS) and Applied AI Engineer interviews at top companies (e.g., FAANG, tier-1 tech startups).
- **Target Knowledge Profile:** Focus on cracking high-bar interviews (systems design, production realities, machine learning intuition, and debugging/refactoring) rather than academic derivations or pedantic schoolbook memorization.

---

## 2. Core Behavioral Rules for the Agent

1. **No Academic Filler:**
   - Skip introductory academic textbook summaries. Focus on the core mechanics, math intuition, and production failure modes that interviewers at top companies actually test.
2. **Focus on Production Trade-offs:**
   - Every mathematical formula must map directly to:
     - An evaluation choice (e.g., ROC-AUC vs. PR-AUC).
     - A troubleshooting scenario (e.g., singular matrices, exploding gradients, perfect separation).
     - A system constraint (e.g., latency budget, CPU vs. GPU, memory footprint).
3. **Notebooks and Code Style:**
   - Notebooks should reflect clean, production-grade Python code utilizing vectorization and Scikit-Learn pipelines.
   - Code should be executable, self-contained, and accompanied by expected logs and structural explanations.
